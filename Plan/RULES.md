# RULES — Правила работы с файлами плана

> **Этот файл читается первым в каждой сессии.**
> Нарушение любого правила недопустимо.

---

## 1. Запрет на полную перезапись файлов

**НИКОГДА не перезаписывай весь файл целиком.**

Допустимые операции:
- Дописать строки в конец секции
- Изменить статус в `PROGRESS.md` (только поле `статус` одной строки)
- Добавить заметку в секцию `## Заметки сессии` в `PROGRESS.md`

Запрещено:
- Заменять содержимое `CONCEPT.md` или `STEPS.md`
- Удалять существующие строки из `PROGRESS.md`
- Переписывать системный промт шага

## 2. Порядок чтения в начале каждой сессии

1. `Plan/RULES.md` — этот файл (прочитать полностью)
2. `Plan/PROGRESS.md` — найти текущий активный шаг
3. `Plan/CONCEPT.md` — секции, релевантные текущему шагу
4. `Plan/STEPS.md` — карточку текущего шага целиком
5. Файлы кода, указанные в карточке шага (только они)

## 3. Проверка перед коммитом

Перед любым изменением кода убедись:
- [ ] Ты прочитал конечную цель из `CONCEPT.md § Конечный результат`
- [ ] Ты прочитал критерии готовности шага из `STEPS.md`
- [ ] Твои изменения не ломают уже реализованные шаги (см. PROGRESS.md)

## 4. Обновление прогресса

После завершения шага **только** обновить строку в `PROGRESS.md`:
```
| STEPN | ... | ⏳ В работе | ... |
```
заменить на:
```
| STEPN | ... | ✅ Готово   | ... |
```
И дописать в секцию `## Лог изменений` одну строку вида:
```
- STEPN завершён: <краткое описание что сделано>
```

## 5. Если что-то непонятно

Если постановка шага неоднозначна — **задай вопрос пользователю**, не интерпретируй самостоятельно.

## 6. Галлюцинации и несуществующий код

- Не придумывай методы, которых нет в `settings_service.py`
- Проверяй импорты — если класс не найден в `models.py`, он не существует
- При сомнении читай файл через инструмент, не полагайся на память
