# Spec-05b: Settings UI — Tabs 1-3 (Domains, Vaults, Generation Models)

Перед выполнением прочитай `Spec-05a` (базовая структура settings.js уже создана).

**Зависит от:** `Spec-05a` (класс SettingsManager, api.js методы).

**Цель:** Реализовать полноценную работу вкладок «Домены», «Vault'ы», «Генеративные модели» в странице настроек.

## Контекст

**Прочитать перед реализацией:**
- `rag-backend/static/js/settings.js` — уже содержит методы-заглушки
- `rag-backend/static/css/chat.css` — добавим стили для карточек и модальных окон

## Задачи

### 1. Реализовать рендеринг вкладки «Домены» в `settings.js`

Метод `renderDomainsTab()` должен возвращать HTML.

**Требования:**
- Список всех доменов (из `api.getDomains()`).
- Для каждого домена карточка: `display_name`, `domain_id`, badge `enabled`/`disabled`, кнопка `Редактировать` (открывает модалку с вкладками «Промпты» и «Поля уточнения»), toggle `enabled/disabled`, кнопка `Удалить` (запрещена для `is_system=true` или если есть vault'ы — бэкенд вернёт 409, но UI может предупреждать).
- Кнопка «+ Новый домен» — модалка: поля `domain_id` (slug, 3-32 символа, `[a-z0-9_]+`), `display_name`, `description` (опционально), чекбокс `enabled`.

**Модалка редактирования домена (вложенные вкладки):**
- Основная вкладка: редактирование `display_name`, `description`, `enabled`.
- Вкладка «Промпты»: textarea для каждого типа (`system`, `clarification`, `planner`, `pipeline_router`). Кнопка «Сохранить».
- Вкладка «Поля уточнения»: список полей с возможностью добавления/удаления. Каждое поле: `field_name` (slug), `label`, `hint`, `required` (чекбокс), `display_order` (число). Кнопка «Сохранить» (полная замена).

**Логика:**
- При сохранении полей уточнения сначала проверить через API. Если бэкенд возвращает 409 (есть активные чаты с этими полями), показать сообщение пользователю.

### 2. Реализовать рендеринг вкладки «Vault'ы»

Метод `renderVaultsTab()`.

**Требования:**
- Список vault'ов из `api.getSettingsVaults()`.
- Для каждого: `display_name` или `vault_id`, домен (название из кэша), путь `/data/vaults/{vault_id}`, embedding-модель (название модели из БД), `binding_status`, `chunk_count`.
- Кнопки: `Редактировать` (модалка), `Вкл/Выкл` (toggle), `Удалить`.
- Кнопка «+ Новый vault».

**Модалка создания/редактирования:**
- Поля: `vault_id` (только при создании, slug 3-64), `domain_id` (select из списка доменов), `display_name` (опционально), `embedding_model_id` (select из списка embedding-моделей), чекбокс `create_folder` (только при создании).
- Per‑vault параметры чанкинга: `chunk_size`, `overlap`, `entity_aware_mode`. Эти поля опциональны — если не заданы, используются глобальные. Плейсхолдеры показывают текущие глобальные значения (брать из `settingsService.getSettingsParams()`).
- При редактировании: если меняется `embedding_model_id`, показать предупреждение: «Это удалит все векторы vault'а и потребует полной реиндексации. Продолжить?» (OK/Cancel).
- При удалении — подтверждение.

### 3. Реализовать рендеринг вкладки «Генеративные модели»

Метод `renderGenerationModelsTab()`.

**Требования:**
- Список моделей из `api.getGenerationModels()`.
- Карточка: `display_name` (или `model_id`), `base_url`, badge `активна`/`не активна`.
- Кнопки: `Проверить доступность` (вызов `api.checkGenerationModel(id)`, показать ✅ latency или ❌ ошибку), `Сделать активной` (вызов `api.activateGenerationModel(id)`, после чего обновить список), `Редактировать`, `Удалить`.
- Кнопка «+ Новая модель».

**Модалка создания/редактирования:**
- Поля: `model_id` (при создании), `display_name` (опционально), `base_url` (валидный URL), `api_key` (password input, кнопка «Очистить»), `timeout_seconds` (int, дефолт 60).
- При сохранении: если `api_key` пустой, не передавать его (оставить старый при редактировании).

### 4. Добавить стили в `css/chat.css`

Добавить классы для:
- `.settings-tabs` и активная вкладка
- `.settings-card` — карточка сущности
- `.settings-card .badge`
- `.settings-card .actions`
- `.modal` (переиспользовать стили из db_management.js, либо создать новые)
- `.modal-tabs`
- `.status-banner` с цветами: `.status-banner.critical` (красный), `.warning` (жёлтый), `.info` (синий/серый)

### 5. Интеграция

В `settings.js` реализовать методы:
- `attachTabEventHandlers(tabId)` — назначить обработчики для кнопок внутри активной вкладки.
- Все операции должны обновлять список после успешного выполнения (перезагрузить текущую вкладку или точечно обновить DOM).

**Обработка ошибок:**
- При `409` (конфликт) показывать сообщение из `error.response.data.detail`.
- При `422` — показывать «Недопустимое значение для поля ...».
- При других ошибках — общее сообщение.

## Финальный контракт

- Вкладки «Домены», «Vault'ы», «Генеративные модели» полностью рабочие.
- Можно создавать, редактировать, удалять (с проверками).
- Для vault'ов работает смена embedding-модели с предупреждением.
- Для генеративных моделей — проверка доступности и активация.

## Критерии приёмки

- [ ] При открытии вкладки «Домены» отображаются все домены.
- [ ] Создание нового домена → появляется в списке без перезагрузки страницы.
- [ ] Редактирование промптов домена — после сохранения обновляется в БД.
- [ ] При удалении домена с vault'ами → ошибка 409, UI показывает сообщение.
- [ ] Вкладка «Vault'ы»: создание vault'а с `create_folder=true` создаёт папку на хосте.
- [ ] При смене embedding-модели в vault'е появляется предупреждение, после подтверждения статус становится `unbound`.
- [ ] Вкладка «Генеративные модели»: проверка доступности показывает ✅ + latency.
- [ ] Активация модели — badge «активна» перемещается на новую модель.
- [ ] Удаление активной модели — кнопка disabled или бэкенд возвращает 409 (UI обрабатывает).
- [ ] Все модалки закрываются по Escape и клику на фон.
