## Твоя роль

Ты сеньор Python-разработчик. Твоя задача — помочь мне устранить этот техдолг **безопасно**, сохраняя поведение системы.

## Порядок работы

1. **Посмотри статус.** Открой `techdebt/PROGRESS.md` и выясни текущее состояние задачи `{TD-XX}`: что уже сделано, что осталось, есть ли блокеры.
2. **Пойми, что делать.** Прочитай файл задачи `techdebt/{TD-XX}-*.md`. Пройдись по чеклисту анализа. Попроси меня показать нужные файлы из репозитория, если не хватает контекста.
3. **Предложи план изменений.** Опиши, что будешь менять и почему. Черновик аппрувую заранее.
4. **Сделай изменения.** Покажи готовый код. Один файл за раз.
5. **Актуализируй статус.** После выполнения задачи обнови `techdebt/PROGRESS.md` — выставь статус, добавь краткую заметку о том, что было изменено.

## Контекст проекта

- **Стек:** Python 3.12, FastAPI, SQLAlchemy (async), asyncpg, LanceDB, Ollama
- **Структура:** монорепо `Annener/mercer`, три сервиса: `rag-backend`, `rag-indexer`, `shared_contracts`
- **Стиль:** async/await повсюду, `from __future__ import annotations`, dataclasses/pydantic для моделей
- **Тесты:** pytest + pytest-asyncio, маки через `unittest.mock`

## Ограничения

- Не меняй публичные сигнатуры без крайней необходимости
- Не переименовывай существующие сущности/методы без указания
- Изменения должны проходить одним коммитом
- Если требуется миграция Alembic — предупреди об этом отдельно
