# TD-01 — Ghost-синглтон PipelineRouter

**Приоритет:** 🔴 Критический  
**Файл:** `rag-backend/app/routers/pipeline_router.py` (последние строки)

## Проблема

В конце файла создаётся экземпляр класса в обход `__init__`:

```python
# Текущий код (ПРОБЛЕМА):
pipeline_router = PipelineRouter.__new__(PipelineRouter)
```

`PipelineRouter.__init__` принимает `db: AsyncSession` как обязательный параметр.
Через `__new__` этот `__init__` **не вызывается**, поэтому у созданного объекта
`self.db = None` (и другие атрибуты не инициализированы).

Если в любом месте кода кто-то импортирует и использует этот синглтон —
будет `AttributeError` в рантайме.

## Анализ перед исправлением

- [ ] Поиск по кодовой базе: `from app.routers.pipeline_router import pipeline_router`
- [ ] Поиск: `pipeline_router.` (использование синглтона)
- [ ] Убедиться, что в тестах нет мока этого объекта
- [ ] Проверить, как `PipelineRouter` инстанциируется в `chat.py` и других местах

## Ожидаемое исправление

Простое: **удалить строку** `pipeline_router = PipelineRouter.__new__(PipelineRouter)`.

Если выяснится, что синглтон где-то используется — переработать на DI (передача
экземпляра через параметры функции / FastAPI `Depends`).
