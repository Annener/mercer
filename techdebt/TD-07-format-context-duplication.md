# TD-07 — Дублирование format_context / format_context_with_role

**Приоритет:** 🟠 Структурный  
**Файл:** `rag-backend/app/services/retrieval.py`

## Проблема

Две функции делают почти одно и то же. Единственное различие — наличие
заголовка `=== role ===`. Логика построения `source_index + numbered` chunks
полностью продублирована между ними.

При изменении формата вывода контекста придётся менять оба места.

## Анализ перед исправлением

- [ ] Посмотреть сигнатуры обеих функций и все места их вызова
- [ ] Убедиться, что единственная разница — наличие/отсутствие заголовка роли
- [ ] Проверить тесты на обе функции

## Ожидаемое исправление

Объединить в одну функцию с опциональным параметром:

```python
def format_context(
    chunks: list[Chunk],
    role: str | None = None,
) -> str:
    header = f"=== {role} ===\n" if role else ""
    # ... единая логика ...
    return header + result
```

Обратная совместимость: `format_context_with_role` становится тонкой обёрткой
или удаляется с обновлением всех вызовов.

## Риски

Низкие. Чисто механический рефакторинг с сохранением поведения.
