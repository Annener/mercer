## Памятка: ключевые архитектурные решения

Чтобы не потерять контекст между чатами — самое важное:

| Решение | Детали |
|---|---|
| Стратегия | Всегда semantic, hardcode в воркере |
| Новое поле в БД | Только `semantic_threshold float DEFAULT 0.3` |
| `preprocess()` | Вызвать ДО передачи в SemanticChunker (на весь текст) |
| `split()` | Принимает уже очищенный текст |
| `embed_batch` | Один вызов на весь документ, не по одному предложению |
| `build_embedding_text` | Вызывается на финальных чанках как раньше, не внутри чанкера |
| LanceDB | Схема не меняется |
| retrieval.py | Не трогать |
| BM25/FTS | Не трогать, работает лучше сам по себе |
| MD-файлы | Работают через тот же SemanticChunker автоматически |
| `word_start` в metadata | Заполнять в SemanticChunker для PDF page assignment |
| Миграции | Кастомный SQL в migrations.py, не Alembic |
| Async-паттерн | CPU-bound через asyncio.to_thread, HTTP через httpx |
