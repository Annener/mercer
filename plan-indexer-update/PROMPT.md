# Промпт для нового чата

> Скопируй всё что ниже горизонтальной линии и вставь в новый чат как первое сообщение.

---

Репозиторий: https://github.com/Annener/mercer (монорепо, Python/FastAPI, Docker Compose).

Мы реализуем план из директории `plan-indexer-update/` — внедрение Redis как единого state-хранилища вместо JSON-файлов и WebSocket.

**Прочитай через GitHub MCP следующие файлы перед тем как начать:**
1. `plan-indexer-update/CONCEPT.md` — архитектура и ключевые решения
2. `plan-indexer-update/STATUS.md` — текущий статус всех этапов (смотри колонку «Статус»)
3. `plan-indexer-update/step-NN-*.md` — файл нужного этапа

**Порядок работы:**
1. Прочитай STATUS.md — найди первый этап со статусом ⬜ (не начат)
2. Прочитай соответствующий step-файл полностью
3. Прочитай все упомянутые в нём исходные файлы репозитория через MCP
4. Реализуй изменения, запушь в ветку main
5. Обнови STATUS.md: поставь ✅, запиши коммит, добавь строку в историю
6. Если в step-файле есть раздел «✅ Unit-тесты» — запусти тесты (попроси пользователя привести нужные файлы или используй MCP)

**Важные детали:**
- `rag-backend` работает с Redis напрямую через `redis.asyncio` — НЕ импортирует `RedisStateManager` из `rag-indexer`
- Все методы `RedisStateManager` — async
- vault-кэш (`vault:{vault_id}:files`) — без TTL; task-state (`task:{task_id}`) — TTL 86400с
- Для unit-тестов используй `fakeredis[aioredis]>=2.0`
- После каждого этапа обязательно обновляй `STATUS.md`
