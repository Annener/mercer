# Mercer — Project Context

Эта папка содержит высокоуровневое описание проекта для работы с ИИ-ассистентами.
Перед тем как просить ИИ внести изменения в проект — укажи ему обратиться к нужному файлу контекста вместо самостоятельного исследования репозитория.

## Структура контекста

| Файл | Описание |
|---|---|
| `architecture.md` | Общая архитектура, сервисы, сеть, порты |
| `db_schema.md` | ORM-модели PostgreSQL, связи, ключевые поля |
| `rag_pipeline.md` | RAG-пайплайн: индексация, ретривал, генерация |
| `api_routes.md` | HTTP API rag-backend: роутеры, эндпоинты, контракты |
| `shared_contracts.md` | Pydantic-схемы из shared_contracts/models.py |
| `conventions.md` | Соглашения по коду, паттерны, именование |

## Как обновлять контекст

При значительных изменениях в проекте (новые сервисы, новые роуты, изменения схемы БД)
обновляй соответствующий файл контекста вместе с кодом.

> Папки `Plan/` и `plan-*` не отражены в контексте — они содержат задания, а не рабочий код.
